Making a Gherkin

Let's imagine we are building an application that allows users to keep a list of contacts with email addresses and phone numbers. As a very first step, we need to define what will happen when a user adds a contact to their contact list. Let's write this as the scenario.

Scenario: A user adds a new contact to their contact list.

So far so good. Next can we define some pre-conditions that are necessary in order for the scenario to take place? Doing this helps us be sure we are not making assumptions about any number of factors that might influence the outcomes of the following action.

For example, we might assume the user has an existing contact list, or that the contact the user is attempting to add does not already exist. How might this affect the outcome of a user adding a contact to their contact list? Often, they resulting behavior should be completely different in each of those scenarios. And will, as a result, likely require their own complete Given, When, Then Scenario. We'll come back to that later. For now let's specify our assumptions in a Given statement.

Given A logged in user with at least one existing contact list.

The When statement defines the action that takes place.

When The user enters at least the minimum required information for a contact whose name does not already exist in the contact list and clicks the add user button.

What should happen? This statement will become the assertions

Then the length of the contact list should increase by 1
Then the newly created user should be present on the list.

Notice that there are two Then statements above. A single Scenario can have multiple Then's, as well as multiple Given's, but usually only one When. However, use discretion with multiple Given's and Then's this leads to better tests.
