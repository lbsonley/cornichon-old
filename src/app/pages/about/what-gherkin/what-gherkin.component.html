What is a Gherkin?

In Behavior Driven Development, A Gherkin is the specification of an individual requirement for a product feature.

A Gherkin could also be described as a scenario. In fact, the first step in creating a Gherkin is to define it's Scenario. The scenario introduces a situation in which the application should perform action or react to user input. The exact behavior of the application for this scenario is written using predefined keywords. Like this:

Given condition or state of the application or user,
When an action is performed on the application,
Then the following outcomes will result.

The combination of a Scenario defined by a Given, When and Then statement is known as a Gherkin.
